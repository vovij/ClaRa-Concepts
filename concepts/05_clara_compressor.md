## How CLaRa Compresses Documents

CLaRa's approach to compression is what makes the rest of its architecture possible. Rather than compressing documents into shorter text, CLaRa compresses them into continuous vector representations — specifically, a small set of learned **memory tokens** appended to each document. When the document is passed through the LLM, these memory tokens attend over the full document content and their final hidden states are taken as the compressed representation. A document of 256 tokens can be compressed down to as few as 1–2 memory tokens depending on the compression ratio, with the hidden states of those tokens encoding the document's salient information in the model's latent space.

The key challenge is making sure these compressed representations actually preserve the right information. CLaRa addresses this through **SCP (Salient Content Preservation)**, a pretraining framework that supervises the compressor using two complementary objectives. The first is QA-based supervision: the compressor must produce representations from which the generator can answer questions about the document, forcing it to retain factually important content rather than surface patterns. The second is paraphrase supervision: the compressor must produce similar representations for paraphrases of the same document, encouraging semantic consistency rather than lexical memorisation.

On top of these two objectives, CLaRa adds an **MSE alignment loss** that minimises the mean squared error between the averaged hidden states of the original document tokens and the memory tokens. This acts as a regulariser, preventing the memory tokens from drifting into an arbitrary region of the latent space — they are explicitly encouraged to summarise the same semantic content as the full document representation. The total pretraining loss is a weighted combination of the cross-entropy generation loss and this alignment term, with a scalar λ balancing the two.

The result is a compressor that produces representations which are both semantically rich and stable — stable being crucial, because in the later training stages the document embeddings are frozen and used as retrieval targets. If the compressed representations were noisy or inconsistent, the entire retrieval mechanism built on top of them would fail.
