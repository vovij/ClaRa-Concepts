## Retrieval Augmented Generation (RAG)

Retrieval-Augmented Generation (RAG) is a framework that enhances Large Language Models by grounding their responses in externally retrieved documents. Rather than relying solely on knowledge encoded in the model's parameters during training, RAG allows the model to query a document store at inference time and condition its output on the retrieved content. This addresses two core weaknesses of standalone LLMs: hallucination (generating plausible but factually incorrect content) and knowledge staleness (being unaware of information beyond the training cutoff).

A standard RAG pipeline has three components. First, an indexing stage processes a document corpus offline — each document is chunked into smaller passages, encoded into dense vector embeddings using a retriever model, and stored in a vector database. Second, at inference time, a retrieval stage takes the user query, encodes it into the same embedding space, and performs a nearest-neighbour search to fetch the top-k most similar document chunks by cosine similarity. Third, a generation stage concatenates the retrieved chunks with the original query as context and passes this to an LLM, which produces the final answer conditioned on both.

The fundamental structural problem with this setup is that the retriever and generator are trained independently with no shared objective. The retriever is optimised for embedding similarity, while the generator is optimised for answer quality — these two objectives are never directly aligned. This means the retriever has no signal about what information the generator actually needs, and the generator has no mechanism to guide what gets retrieved. This disjoint optimisation is the gap that CLaRa aims to close.
