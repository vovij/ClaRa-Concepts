## CLaRa's Three-Stage Training

CLaRa implements three LoRA adapters on a single base LLM: a **compressor** (θc), a **query reasoner** (θqr), and a **generator** (θg). These are trained across three stages, each building on the last.

**Stage 1 — Compressor Pretraining.** Only the compressor adapter is trained. Using the SCP objectives described above (QA supervision + paraphrase supervision + MSE alignment), the compressor learns to distil documents into memory token representations that are semantically faithful and retrievable. At the end of this stage, the compressor is frozen — its weights are fixed for all subsequent stages, and documents can be encoded offline into a static index of compressed embeddings.

**Stage 2 — Instruction Tuning.** The compressor remains frozen. Both the query reasoner and the generator adapters are fine-tuned on instruction-following QA tasks, where the input is a compressed document representation paired with a task instruction, and the target output is a reference answer generated by a teacher model. This stage adapts the generator to consume continuous compressed representations rather than raw text, and gives the query reasoner a starting point in the right representational space.

**Stage 3 — End-to-End Joint Training.** This is the core of CLaRa. The query reasoner and generator are trained jointly using a single language modelling loss on the final answer. The query reasoner encodes the input query into the same latent space as the compressed document embeddings, retrieval is performed by cosine similarity, and the top-k documents are selected and passed to the generator. Crucially, the top-k selection — normally a discrete, non-differentiable operation — is made differentiable using a straight-through estimator, allowing gradients from the generation loss to flow back through the retrieval step into the query reasoner. This means the query reasoner is implicitly trained to retrieve documents that make the generator more likely to produce the correct answer, without any explicit retrieval supervision labels.
